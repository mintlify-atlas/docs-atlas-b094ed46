This guide will walk you through setting up Resonance on your local machine. You'll have a fully functional text-to-speech platform running in under 15 minutes.

## Prerequisites

Before you begin, make sure you have the following:

<Steps>
  <Step title="Node.js 20.9 or later">
    Download from [nodejs.org](https://nodejs.org) or use a version manager like `nvm`.
    
    ```bash
    node --version  # Should be 20.9.0 or higher
    ```
  </Step>
  
  <Step title="Prisma Postgres Database">
    Sign up for a free database at [Prisma Data Platform](https://cwa.run/prisma). You'll need the connection string.
    
    <Info>Any PostgreSQL database will work - Prisma Postgres is recommended for its simplicity.</Info>
  </Step>
  
  <Step title="Clerk Account (with Organizations enabled)">
    Create an account at [clerk.com](https://cwa.run/clerk) and enable Organizations in your application settings.
  </Step>
  
  <Step title="Cloudflare R2 Bucket">
    Set up a bucket at [Cloudflare R2](https://cwa.run/cloudflare-r2). The free tier includes 10GB storage.
  </Step>
  
  <Step title="Modal Account">
    Sign up at [modal.com](https://cwa.run/modal) for serverless GPU hosting. Install the CLI:
    
    ```bash
    pip install modal
    modal setup
    ```
  </Step>
  
  <Step title="Polar Account (Optional)">
    For billing features, create an account at [polar.sh](https://cwa.run/polar). Use sandbox mode for testing.
  </Step>
</Steps>

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/code-with-antonio/resonance.git
    cd resonance
    ```
  </Step>
  
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
    
    This will also run `prisma generate` automatically via the postinstall script.
  </Step>
  
  <Step title="Configure environment variables">
    Create a `.env.local` file in the root directory with the following variables:
    
    ```env .env.local
    # Database
    DATABASE_URL="postgresql://user:password@host:5432/database"
    
    # Chatterbox TTS (Modal)
    CHATTERBOX_API_URL="https://your-modal-endpoint.modal.run"
    CHATTERBOX_API_KEY="your-api-key"
    
    # Clerk
    CLERK_SECRET_KEY="sk_..."
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_..."
    NEXT_PUBLIC_CLERK_SIGN_IN_URL="/sign-in"
    NEXT_PUBLIC_CLERK_SIGN_UP_URL="/sign-up"
    
    # Cloudflare R2
    R2_ACCOUNT_ID="your-account-id"
    R2_ACCESS_KEY_ID="your-access-key"
    R2_SECRET_ACCESS_KEY="your-secret-key"
    R2_BUCKET_NAME="your-bucket-name"
    
    # Polar (Billing)
    POLAR_ACCESS_TOKEN="your-polar-token"
    POLAR_SERVER="sandbox"
    POLAR_PRODUCT_ID="your-product-id"
    
    # App
    APP_URL="http://localhost:3000"
    ```
    
    <Warning>Keep your `.env.local` file secure and never commit it to version control.</Warning>
  </Step>
  
  <Step title="Set up the database">
    Run the Prisma migrations to create the database schema:
    
    ```bash
    npx prisma migrate deploy
    ```
    
    This creates two main tables:
    - `Voice` - Stores system and custom voice metadata
    - `Generation` - Stores TTS generation history and parameters
  </Step>
</Steps>

## Deploy the TTS Engine

The TTS engine runs on Modal's serverless GPU infrastructure. You need to deploy it before the web app can generate audio.

<Steps>
  <Step title="Configure R2 credentials in chatterbox_tts.py">
    Open `chatterbox_tts.py` and update the R2 configuration:
    
    ```python chatterbox_tts.py
    R2_BUCKET_NAME = "your-bucket-name"
    R2_ACCOUNT_ID = "your-account-id"
    ```
  </Step>
  
  <Step title="Create Modal secrets">
    Create the required secrets in your [Modal dashboard](https://modal.com/secrets):
    
    | Secret Name | Keys | Description |
    |-------------|------|-------------|
    | `cloudflare-r2` | `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY` | R2 API credentials |
    | `chatterbox-api-key` | `CHATTERBOX_API_KEY` | API key to protect the endpoint (use any strong random string) |
    | `hf-token` | `HF_TOKEN` | Hugging Face token for downloading model weights |
    
    Or create them via CLI:
    
    ```bash
    modal secret create cloudflare-r2 \
      AWS_ACCESS_KEY_ID=your-r2-access-key \
      AWS_SECRET_ACCESS_KEY=your-r2-secret-key
    
    modal secret create chatterbox-api-key \
      CHATTERBOX_API_KEY=your-strong-random-string
    
    modal secret create hf-token \
      HF_TOKEN=your-huggingface-token
    ```
  </Step>
  
  <Step title="Deploy to Modal">
    ```bash
    modal deploy chatterbox_tts.py
    ```
    
    This deploys Chatterbox TTS to a serverless NVIDIA A10G GPU. The container mounts your R2 bucket read-only for direct access to voice reference audio.
    
    Copy the resulting Modal URL and set it as `CHATTERBOX_API_URL` in your `.env.local`.
    
    <Info>The first request after inactivity may take 10-15 seconds due to cold starts as Modal provisions the GPU container.</Info>
  </Step>
  
  <Step title="Generate the API client">
    Once deployed, generate the type-safe Chatterbox client from the OpenAPI spec:
    
    ```bash
    npm run sync-api
    ```
    
    This creates TypeScript types in `src/types/chatterbox-api.ts` for full type safety when calling the TTS API.
  </Step>
</Steps>

## Seed System Voices

Resonance includes 20 pre-built voices across different categories and locales. Seed them to your database and R2 bucket:

```bash
npx prisma db seed
```

This uploads the voice samples from the repository (included WAV files from [Modal's voice sample pack](https://modal-cdn.com/blog/audio/chatterbox-tts-voices.zip)) and creates the corresponding database records.

<Note>
The seed script creates voices in categories like Audiobook, Conversational, Narrative, Characters, Meditation, Podcast, Advertising, Voiceover, and Corporate.
</Note>

## Run the Application

Start the development server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser. You'll be prompted to sign in with Clerk.

<Steps>
  <Step title="Sign up and create an organization">
    Create a new account and set up your first organization. All voices and generations are scoped to organizations.
  </Step>
  
  <Step title="Generate your first audio">
    Navigate to the Text-to-Speech page, select a voice, enter some text, and click Generate. Your audio will be created in seconds!
  </Step>
  
  <Step title="Clone a voice (optional)">
    Go to the Voices page and click "Create Voice". Upload a 10+ second audio sample or record one directly in the browser.
  </Step>
</Steps>

## Testing with Sandbox Mode

To test billing features without real payments, use Polar's sandbox mode:

1. Set `POLAR_SERVER="sandbox"` in your `.env.local`
2. Use the test card number `4242 4242 4242 4242` with any future expiration date and any CVC
3. All charges will be simulated - no real money will be processed

<Warning>
Remember to switch to `POLAR_SERVER="production"` when deploying to production!
</Warning>
